# Default ProcessMessage Convention

`Control::ProcessMessage` 的默认约定是：控件只处理自己明确关心的消息，并把未消费的消息交回父类或容器继续分发。

## 返回值

- 返回 `true`：消息已被当前控件消费，调用方不应继续按普通控件链路处理。
- 返回 `false`：当前控件没有消费消息，调用方可以继续做默认处理或尝试其他目标。

## 推荐顺序

1. 先检查控件是否启用、可见，以及消息是否落在控件有效区域。
2. 处理当前控件自己的交互状态，例如按下、悬停、拖拽、键盘编辑。
3. 触发对应事件，例如 `OnMouseClick`、`OnKeyDown`、`OnTextChanged`。
4. 对容器控件，优先把输入消息转发给更具体的子控件。
5. 未处理的消息交给基类实现。

## 设计器注意事项

设计器画布会主动收敛键盘输入、选择、拖拽、缩放和撤销重做入口。修改控件消息处理时，要确认不会绕过设计器的 `DesignerCanvas::ProcessMessage` 交互快照，否则 Undo/Redo 可能无法记录完整操作。
